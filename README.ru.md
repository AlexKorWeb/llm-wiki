# LLM Wiki 🧠

**Самоподдерживающаяся база знаний, где индекс — это LLM, а не векторная база.**
Пересылаешь пост из Telegram, кладёшь заметку или вставляешь URL → AI-агент исследует тему, пишет подробный источник и связанную вики-страницу, обновляет навигационный индекс и коммитит в git. Под ключ.

Вдохновлено [методом ведения заметок Андрея Карпатого](https://github.com/karpathy): вместо RAG по чанкам — чистая, вручную выверенная Markdown-вики, которую модель сама читает, дополняет и связывает.

🇬🇧 [English README](README.md)

---

## Зачем

- **Без векторной БД, эмбеддингов и инфраструктуры.** Только Markdown + git + LLM.
- **Вики остаётся читаемой человеком** — открой в любом редакторе, Obsidian или на GitHub.
- **Ссылки не теряются.** Каждая страница цитирует канонические URL (агент находит их, если пост их вырезал).
- **Приём без рук.** Telegram-бот превращает «переслал и забыл» в готовые закоммиченные статьи.
- **Портативность как Skill.** Встроенный Claude Code Skill можно подключить к любому проекту.

## Как это работает

```text
              Telegram (переслать пост / вставить URL)
                              │
                              ▼
                   scripts/tg_bot.py   ──сохраняет──►  raw/_inbox/<ts>_<источник>.md
                              │                         (+ картинки, + YouTube-транскрипт)
                  (debounce ~60с тишины)
                              │
                              ▼
                   claude -p  (один прогон на файл)
                              │
        ┌─────────────────────┼─────────────────────────────┐
        ▼                     ▼                              ▼
  raw/<тема>.md       knowledge/<кат>/<стр>.md          index.md + log.md
  (подробно,          (кратко, URL-источники,           (навигация +
   обогащено через     связи через related:)             append-only история)
   WebSearch)
                              │
                              ▼
                   git add + commit + push   ──►   уведомляет тебя в Telegram
```

Метод описан в [`CLAUDE.md`](CLAUDE.md) (агент читает его первым) и во встроенном
[`.claude/skills/llm-wiki/SKILL.md`](.claude/skills/llm-wiki/SKILL.md).

## Структура

```text
.
├── CLAUDE.md             # метод (INGEST / QUERY / LINT) — агент читает первым
├── .claude/skills/llm-wiki/SKILL.md   # устанавливаемый Claude Code Skill
├── raw/                  # источники (read-only истина)
│   ├── _inbox/           # сюда бот кладёт пересланные посты (в .gitignore)
│   └── _assets/          # картинки, ставшие частью базы знаний
├── knowledge/
│   ├── concepts/         # понятия, методики, техники
│   ├── tools/            # утилиты, библиотеки, сервисы, CLI, платформы
│   ├── skills/           # только Claude Code Skills
│   └── connections/      # связи между темами (X vs Y, X + Y, паттерны)
├── daily/                # автоматические логи сессий
├── scripts/              # tg_bot.py, flush.py, compile.py, .env.example
├── index.md              # навигационный индекс (обновляется при каждом INGEST)
└── log.md                # append-only история операций (новое сверху)
```

## Требования

- **[Claude Code](https://claude.com/claude-code)** установлен и авторизован (`claude` в `PATH` — бот вызывает `claude -p`).
- **Python 3.10+** (для Telegram-бота).
- **git** + репозиторий на GitHub (бот сам коммитит и пушит после каждого INGEST).
- **Токен бота** от [@BotFather](https://t.me/BotFather) и твой числовой user ID от [@userinfobot](https://t.me/userinfobot).

## Быстрый старт

### 1. Используй шаблон

Нажми **«Use this template»** на GitHub (или `git clone`), затем нацель на свой приватный репо:

```bash
git clone https://github.com/AlexKorWeb/llm-wiki.git my-wiki
cd my-wiki
```

> 💡 Держи **свою** вики приватной — заметки личные. Этот репозиторий — только движок.

### 2. Попробуй вручную (без бота)

Положи любую заметку в `raw/`, открой Claude Code в репозитории и скажи:

```text
Обработай raw/my-note.md
```

Claude прочитает `CLAUDE.md`, исследует тему и создаст `knowledge/`-страницу + обновит `index.md` и `log.md`.

Можно просто задавать вопросы (QUERY) или просить проверку (LINT):

```text
Что у нас по теме X?         # QUERY — отвечает из вики, хорошие ответы сохраняет как страницы
Проверь вики (lint)          # LINT — битые ссылки, орфаны, противоречия, устаревшее
```

### 3. Настрой Telegram-бота (приём без рук)

```bash
cd scripts
pip install -r requirements.txt
cp .env.example .env          # затем отредактируй .env
```

Заполни `scripts/.env`:

```env
BOT_TOKEN=123456789:AA-твой-токен-бота
OWNER_USER_ID=000000000       # только ты можешь писать боту
DEBOUNCE_SECONDS=60           # подождать столько после последнего поста, потом INGEST
CLAUDE_MAX_TURNS=60
```

Запусти:

```bash
python scripts/tg_bot.py
```

Теперь **пересылай любой пост** (или отправляй URL / YouTube-ссылку / картинку с подписью) боту. Через ~60с тишины он прогонит `claude -p` по каждому элементу, напишет статьи, закоммитит, запушит и пришлёт сводку. Команды: `/status`, `/stop` (плюс inline-кнопка 🛑).

## Obsidian и синхронизация с телефоном 📱

Вики — это обычный Markdown, поэтому она открывается как **[Obsidian](https://obsidian.md)-vault**: *Open folder as vault* → укажи папку репозитория.

- **Graph view** визуализирует сеть знаний: каждая запись `related:` и каждый `[[викилинк]]` становится ребром графа, а `index.md` — центральным узлом. Смотреть, как граф растёт после каждого INGEST — отдельное удовольствие.
- **Backlinks, полнотекстовый поиск и теги** работают из коробки по всем страницам `knowledge/`.

### Читать и править с телефона через GitHub

Telegram-бот уже коммитит и пушит после каждого INGEST, поэтому телефону остаётся только **pull**:

1. Установи **Obsidian** на телефон и открой клонированный репозиторий как vault.
2. Установи community-плагин **[Obsidian Git](https://github.com/Vinzent03/obsidian-git)** (на Android работает напрямую; на iOS — в связке с [Working Copy](https://workingcopy.app)).
3. Включи **«Pull on startup»** (и, по желанию, авто-pull по интервалу). Теперь каждая статья, которую сгенерировал бот, появляется на телефоне — вместе с графом и backlinks.

Полный цикл:

```text
переслал пост с телефона
        │
        ▼
бот делает INGEST на всегда-включённой машине  →  git commit + push на GitHub
        │
        ▼
Obsidian Git делает pull на телефоне  →  читаешь готовую связанную статью
```

> `.obsidian/` в `.gitignore`, поэтому у каждого устройства свой layout и плагины — поставь Obsidian Git один раз на каждом устройстве. (Хочешь общие цвета/настройки графа? Убери `.obsidian/` из `.gitignore` и закоммить минимальный конфиг.)

## Работа 24/7 (опционально)

Бот работает, пока жив процесс и доступен `claude`. Чтобы держать его всегда включённым:

- **Локально:** под менеджером процессов (pm2, nssm на Windows, systemd на Linux) или в `tmux`/screen.
- **На сервере:** положи репозиторий на маленький VPS с установленным Claude Code; синхронизация через git.

## Безопасность

- **`scripts/.env` в `.gitignore`** — токен бота не попадёт в git. Коммитится только `.env.example` (плейсхолдеры).
- Бот **заблокирован на владельца**: игнорирует всех, кроме `OWNER_USER_ID`.
- `raw/_inbox/` в `.gitignore` (транзитные входящие посты); обработанные картинки переезжают в `raw/_assets/` и трекаются.
- Агенту запрещено редактировать/удалять `raw/` и нарушать append-only порядок `log.md`.

## Кастомизация

- **Язык:** по умолчанию русская проза + английский код/термины. Поменяй строку про язык вверху `CLAUDE.md`.
- **Категории:** редактируй папки `knowledge/` и список секций в `index.md`.
- **Поведение INGEST:** полный промпт на файл — в `scripts/tg_bot.py` (`run_ingest`). Подстрой под себя.
- **Skill:** скопируй `.claude/skills/llm-wiki/` в `~/.claude/skills/`, чтобы запускать операции вики из любого проекта.

## Вспомогательные скрипты

- `scripts/flush.py "описание" [--files a.md b.md]` — добавляет запись о сессии в `daily/YYYY-MM-DD.md` (удобно как Claude Code hook).
- `scripts/compile.py --days 30` — сводка по дневным логам и подсветка страниц, которые стоит обновить.
